# 模型自动自检设计

## 目标

在 Codex Compass 运行期间，按固定周期对所有可检测的供应商模型发送最小真实请求，尽早发现模型、密钥、协议或上游服务不可用。功能默认关闭；用户开启后立即检测一次，之后每 10 分钟检测一次。

## 范围

- 使用 Tauri Rust 后端定时器，不依赖当前打开的前端页面。
- 检测所有配置完整的非聚合 API 供应商。
- 每个供应商优先使用 `testModel`，未填写时回退到该供应商默认模型，再回退到全局 `relayTestModel`。
- 聚合供应商不单独检测，其成员供应商会按普通供应商规则检测。
- 仅使用官方账号且没有 API Key 的供应商标记为“已跳过”。
- Compass 退出后停止检测，不安装 Windows 服务或计划任务。
- 不保存响应正文、API Key 或完整请求内容。

## 设置

在后端设置和“供应商配置”页面增加：

- `modelHealthCheckEnabled: boolean`，默认 `false`。
- 固定检测间隔为 10 分钟，本版本不提供自定义间隔。

开启开关时：

1. 持久化设置。
2. 唤醒后端调度器并立即执行一轮检测。
3. 后续每 10 分钟执行一轮。

关闭开关时，后端取消后续检测；已经发出的单轮请求允许正常结束，但结果不再触发状态通知。

## 后端架构

新增独立的模型健康检查管理器，由 Tauri 应用状态持有。管理器负责：

- 启动后读取最新设置。
- 在功能启用时执行启动检测。
- 通过 10 分钟定时器触发后续检测。
- 每轮开始前重新读取设置，确保供应商、模型和开关修改立即生效。
- 使用互斥状态阻止检测轮次重叠。
- 限制供应商检测并发，避免同时向大量上游发请求。
- 复用现有 `relay_config::test_relay_profile` 真实请求实现。
- 保存本次运行期间的最近结果和上一状态，用于状态转换判断。

后端暴露三个 Tauri 命令：

- `get_model_health_status`：返回当前配置、运行状态、最近结果和上次检测时间。
- `run_model_health_check_now`：手动触发一轮检测；已有检测运行时返回当前状态，不重复启动。
- `set_model_health_check_enabled`：持久化开关并启动或停止调度。

后端通过 Tauri 事件发送状态变化：

- `model-health-check:failed`
- `model-health-check:recovered`
- `model-health-check:updated`

## 供应商筛选

当供应商配置总开关关闭时，不发起检测，状态显示“已暂停”。

单个供应商按以下顺序处理：

1. 聚合供应商：跳过，成员供应商单独检测。
2. 官方账号且没有 API Key：跳过。
3. Base URL、API Key 或最终测试模型为空：标记为“配置不完整”，不发请求。
4. 其他供应商：发送最小真实模型请求。

供应商结果状态为：

- `available`：请求成功且 HTTP 状态码小于 400。
- `unavailable`：网络错误、超时或 HTTP 状态码大于等于 400。
- `skipped`：按规则无需或无法检测。

## 通知规则

首次建立状态基线时：

- 首次成功不弹通知。
- 首次失败弹出失败通知。
- 跳过状态不弹通知。

后续轮次只在以下状态转换时通知：

- `available -> unavailable`：提示供应商和模型不可用，并显示精简错误。
- `unavailable -> available`：提示供应商和模型已恢复。

连续失败或连续成功不重复提醒。错误摘要最多保留 160 个字符，并移除可能包含敏感字段的响应正文。

## 前端界面

在“供应商配置”页增加一块紧凑的模型自检区域：

- “模型自动自检”开关。
- 固定间隔说明“启动后立即检测，之后每 10 分钟检测”。
- “立即检测”按钮。
- 最近检测时间。
- 可用、不可用、跳过数量。
- 每个供应商的模型、状态和简短说明。

状态变化使用现有悬浮通知组件，不推动页面内容。前端监听后端事件，并在页面加载时调用 `get_model_health_status`，因此切换页面不会丢失当前运行状态。

## 并发与错误处理

- 同一时间只允许一轮检测。
- 单轮最多并发检测 3 个供应商。
- 单个供应商沿用现有请求超时。
- 某个供应商失败不会中断其他供应商。
- 设置读取失败时终止本轮并记录管理器错误，不输出敏感配置。
- 前端事件监听失败不影响后端继续检测，重新进入页面后可读取最新状态。

## 测试

- 10 分钟周期计算。
- 供应商筛选和模型回退顺序。
- 聚合与官方账号供应商跳过。
- 检测轮次不重叠。
- 首次失败、故障和恢复通知转换。
- 连续相同状态不重复通知。

- 获取状态。
- 手动触发。
- 开关持久化。

前端测试覆盖：

- 默认关闭显示。
