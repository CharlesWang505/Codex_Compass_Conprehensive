# 手机远控安全模型与威胁分析

## 1. 安全定位

当前版本是免注册、单用户自部署安全 MVP，不是完整的生产级多租户远控平台。它适合用户在自己控制的 HTTPS 网站上，将手机连接到自己的电脑；已经支持电脑确认的一次性局域网配对，但不应宣称具备每设备长期身份、云端恢复或企业设备管理。

## 2. 信任边界

- 电脑端：最终信任边界，持有 Codex 本地认证、工作区授权和执行能力。
- 手机端：持有房间 Token 和内容加密密钥，能提出远程请求。
- 中继：不受信任的密文路由器，可见房间/设备 ID、在线状态、时间、大小和路由元数据。
- Codex app-server：仅在电脑本地通过 stdio 连接。

Codex `auth.json`、OAuth Token、Cookie、API Key 和 8787 上游凭据都不得发送给手机或中继。

## 3. 已实现控制

- 功能默认关闭，关闭时不连接中继。
- 公网地址强制 HTTPS/WSS；明文仅允许 localhost 开发。
- AES-256-GCM 端到端内容加密，路由字段纳入 AAD。
- 每帧随机 nonce、外层 sequence 和 2048 条防重放窗口。
- 内层命令按手机设备隔离幂等，防止换新密文重复执行同一任务。
- 中继按房间 Token 摘要、设备角色和目标设备隔离路由。
- 每连接每分钟 600 条消息限流，单帧最大 512 KiB。
- 附件在浏览器端独立加密，中继只保存限时密文；下载令牌一次有效，电脑端验证大小、SHA-256、工作区路径和文件类型。
- 默认无执行工作区；拒绝相对路径、`..`、磁盘根目录和整个用户主目录。
- 会话列表和历史只返回 Codex 左侧正式项目；未授权项目只读，继续会话和执行仍要求电脑端工作区授权。
- 手机选择模型必须通过本机 app-server 模型目录校验。
- 手机页面使用 `textContent` 渲染消息，不执行不可信 HTML。
- 紧急暂停拒绝新远程命令，本地 Codex 使用不受影响。
- app-server 使用隐藏子进程；中继或 app-server 故障不暴露 8787。
- 局域网配对默认关闭，仅允许可信私网来源；支持电脑邀请和手机主动请求。
- 配对使用 X25519 临时密钥交换、HMAC-SHA256 邀请验证、AES-256-GCM 凭据加密、双端六位校验码、两分钟过期和来源 IP 限流。
- 局域网二维码只包含一次性邀请，不包含 Relay 长期 Token 或内容密钥。
- Windows 远控 app-server 使用进程级 `unelevated` Codex 沙箱覆盖，继续按工作区设置
  `workspace-write` 或 `read-only`；不会修改用户全局配置，也不会回退到
  `danger-full-access`。

## 4. 当前限制

以下能力尚未实现，部署者必须知情：

- 采用免注册设计，当前尚未实现持久化设备绑定数据库、设备恢复码或管理员设备管理。
- 每台手机尚无独立 X25519 长期密钥、签名身份或自动密钥轮换。
- 没有持久化加密离线队列、跨中继重启断点续传或撤销 Token 数据库。
- 敏感桌面文件与普通设置分目录保存，但尚未接入 Windows Credential Manager/DPAPI。
- 公网快速链接仍包含预共享 Token 和内容密钥；局域网配对已避免在二维码中放入长期密钥，但批准后的手机仍共享当前房间密钥，尚不能单独轮换某一台手机。
- 工作区授权覆盖规范化后的根目录及其子目录，canonicalize 后逃逸根目录的符号链接不会获得权限；尚未提供子目录细粒度或敏感文件例外审批。附件只允许写入工作区内固定的 `.codex-compass/uploads/<session>` 目录。
- 附件当前使用房间预共享内容密钥，不是每台手机独立派生密钥；中继可见上传密文大小、时间和路由元数据。
- 未完成手机端高风险审批流程。未知或高风险审批应由电脑端拒绝，不允许仅凭手机批准。
- 暂停会拒绝新命令，但当前版本不会可靠冻结已经交给 app-server 执行的所有任务。
- 中继 ACK 只表示路由投递，不代表任务执行成功。

因此，当前版本只应部署在单用户、可信手机、可信域名和可信反向代理环境中。

## 5. 主要威胁

| 威胁 | 当前缓解 | 剩余风险 |
| --- | --- | --- |
| 中继数据库或主机泄露 | 中继无数据库且只见密文；内容使用 AES-GCM | 元数据、流量大小和在线时间可见 |
| 网络窃听或篡改 | HTTPS/WSS + AES-GCM AAD | 错误配置 TLS 或配对链接泄露仍危险 |
| 重放同一帧 | sequence、消息 ID、防重放窗口 | 桌面重启后外层内存窗口清空 |
| 换 nonce 重发同一命令 | 内层 `messageId` 响应缓存 | 缓存仅在桌面进程生命周期内，最多 2048 条 |
| 跨用户/跨房间转发 | 房间 Token 摘要、角色和目标校验 | 尚未升级为每手机独立设备身份 |
| 任意路径访问 | 电脑端白名单与 canonicalize | 更细敏感文件策略尚未完成 |
| XSS 读取配对密钥 | 严格 CSP、纯文本渲染、fragment 不上送 | 同源服务被攻陷仍可读取浏览器 fragment/存储 |
| 暴力连接 | 256 位随机 Token、连接限流 | 没有 IP/账号级持久化封禁 |
| 高风险命令 | 工作区限制，未知审批拒绝 | Codex 自身工具策略仍需持续验证 |
| 局域网中间人 | X25519、邀请 HMAC、双端校验码、电脑确认 | 手机主动请求若用户不核对校验码，仍可能被同网攻击者转接 |

## 6. 部署要求

- 只公开 443；禁止公开 4178、8787、app-server 或调试端口。
- 使用可信 CA 证书，不接受无效证书。
- 优先使用局域网一次性配对；公网快速链接只在自己的电脑和手机之间传输，不发送到聊天群、邮件归档或工单系统。
- 4179 只允许专用网络，不得配置路由器端口转发或云隧道公开暴露。
- 手机丢失、链接泄露或服务器被入侵时，立即关闭远控；在支持密钥轮换前删除远控敏感配置并重新初始化。
- 不在代理或 Node 日志中记录完整 URL、认证帧、WebSocket payload 或请求头。
- 定期运行依赖审计和 app-server 兼容探针。

## 7. 生产强化路线

进入生产级发布前至少需要：X25519/HKDF 每设备长期密钥、设备签名挑战、DPAPI/Credential Manager、密钥轮换和撤销、持久化设备绑定与恢复码、持久化限流审计、锁屏自动暂停、细粒度风险审批以及独立安全审计。免注册模式以设备身份代替网站账号，不应退化为无认证访问。
