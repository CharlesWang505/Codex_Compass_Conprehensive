# 更新日志

本项目采用语义化版本。每次代码、界面或构建配置修改都必须新增一条版本记录，并同步递增应用版本号。

## [1.3.19] - 2026-07-16

### Relay 一键自部署

- Relay 包增加 Windows 上传向导，用户填写自己的 VPS、域名、邮箱和 SSH 用户后，可自动上传并执行远端安装。
- VPS 安装器支持 Ubuntu/Debian x64 与 arm64，自动安装并校验 Node.js、部署版本目录、配置 systemd、Nginx、Certbot 和 HTTPS/WSS。
- Relay 升级改为独立版本目录和原子链接切换，新服务启动失败时恢复上一版本；已有证书升级时不撤销当前 HTTPS 配置。
- 增加 `codex-compass-relay-doctor` 诊断命令，检查服务、本地健康、Nginx、证书、公网 HTTPS、回环监听和脱敏日志。
- 一键安装不会启用防火墙或开放 4178、4179、8787，只在 UFW 已启用时放行 80/443。
- 新增部署脚本自动化测试和 Windows PowerShell dry-run 验证。

### 自建边界

- 安装器只输出用户自己的 HTTPS/WSS 地址，不包含维护者私人域名、IP 或凭据。
- Compass 新安装默认仍关闭远控且地址为空，用户必须自行部署并填写 Relay。

## [1.3.13] - 2026-07-15

### 局域网配对

- 手机远控增加默认关闭的内嵌 Rust 局域网配对服务，默认端口 4179，仅允许回环、RFC1918、链路本地、IPv6 ULA 和可选 Tailscale 网段。
- 支持电脑生成二维码/六位码邀请手机，也支持手机打开局域网页主动向电脑请求绑定；两种方式都必须由电脑端核对双端六位校验码并批准。
- 配对使用 X25519 临时密钥交换、HMAC-SHA256 邀请验证和 AES-256-GCM 凭据加密；二维码不包含 Relay 长期 Token 或内容密钥。
- 邀请和请求两分钟过期、一次有效，并按来源 IP 限流；取消、拒绝和批准状态会实时同步到手机。
- 局域网页使用经过审计的 Noble 加密实现，在手机通过 `http://电脑局域网IP` 访问且 WebCrypto 不可用时仍可完成标准加密。
- 局域网服务只用于设备配对，不转发 Codex 消息，不开放 8787、app-server 或 Codex stdio。

### VPS 自部署

- 增加使用示例域名和文档保留 IP 的 Ubuntu 22.04 自部署说明，以及独立 Relay `package.json` 和锁文件。
- 增加 systemd 沙箱服务、Nginx HTTPS/WSS 配置、Caddyfile、环境变量模板、安装脚本和部署包生成脚本。
- Relay 生产环境固定监听 `127.0.0.1:4178`，公网只开放 80/443；Nginx 配置包含 WebSocket Upgrade、11 MiB 上限、TLS 与握手限流。
- Relay 空房间只能由电脑设备创建，防止手机在服务重启后抢先占用已知房间 ID。

### 验证

- Rust 局域网配对测试覆盖私网分类、X25519 密钥一致性、HMAC 防篡改、AES-GCM 加解密、桌面批准、一次性邀请、过期、限流和内嵌服务静态页面。
- Playwright 覆盖电脑二维码邀请和手机主动请求，验证纯 HTTP 局域网页可加密配对并安全跳转到公网远控站。
- 前端生产构建、Lint、Relay 测试和局域网配对测试通过。
